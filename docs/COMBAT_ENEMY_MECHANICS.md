# Механика боя и врагов

Документ описывает **фактическую** реализацию на текущий момент (по коду `handlers/combat.py`, `models/player.py`, `models/enemies.py`, `models/classes.py`, `handlers/commands.py`, `handlers/location.py`).

## 1. Где находится логика

- Основной цикл боя: `handlers/combat.py`
- Блокировки команд во время боя: `handlers/commands.py`
- Базовые шаблоны врагов по локациям: `models/enemies.py`
- Статы игрока и производные формулы: `models/player.py`
- Классы и скиллы: `models/classes.py`
- Бои в коридоре перехода: `handlers/location.py` (`_maybe_trigger_travel_event`)

## 2. Когда начинается бой

Бой запускается в нескольких сценариях:

1. Исследование локации (`_complete_research` -> `_handle_research_event` -> `_spawn_enemy`).
2. Событие в пути/коридоре (`_maybe_trigger_travel_event` -> `_spawn_enemy`).
3. Некоторые случайные события (например кровавый след может принудительно запустить бой).

После запуска бой сохраняется в `_combat_state[user_id]` и считается активным, пока не будет:
- победа (`_handle_victory`)
- бегство (`handle_combat_flee`, успешный шанс)
- смерть (`_handle_death`)

## 3. Что можно во время боя

Из `handle_combat_commands`:
- Разрешено: `Атаковать`, `Навыки`, `Инвентарь`, `Убежать`, использование расходников.
- Запрещены переходы по локациям/экранам (город, КПП, рынок, персонаж и т.д.).
- При любом постороннем вводе игроку снова показывается боевая клавиатура.

То есть бой имеет приоритет над навигацией UI.

## 4. Как формируется враг

### 4.1 Базовый пул врагов

В `models/enemies.py` для каждой исследуемой зоны есть список врагов с базовыми `hp`, `damage`, `chance`.

Локации:
- `дорога_военная_часть`
- `дорога_нии`
- `дорога_зараженный_лес`

### 4.2 Скалирование под игрока (`_scale_enemy_for_player`)

Враг не берется "как есть", а масштабируется:

1. Вычисляется целевой уровень врага:
- `desired_level = round(zone_level * 0.35 + player_level * 0.75)`
- добавляется случайный разброс (`spread` до 5)
- затем зажимается в диапазон:
  - минимум: `max(1, int(player_level * 0.55) - 2)`
  - максимум: `max(zone_level + 6, int(player_level * 1.15) + 4)`

2. Множители роста:
- HP: `1.0 + 0.035 * (enemy_level - 1)`
- Урон: `1.0 + 0.028 * (enemy_level - 1)`

3. Назначается роль врага:
- `bruiser` (танк): больше HP, меньше урон
- `assassin` (хищник): меньше HP, выше урон, шанс уклонения
- `controller` (контролёр): дренаж энергии при попадании

4. Возможна элитная версия:
- открывается с 12+ уровня игрока
- шанс растёт мягко до 18%
- элитка получает доп. множители HP/урона и повышенный reward multiplier

5. Защита ранней игры:
- для игрока `level <= 10` действует верхний потолок для HP/урона врага, чтобы не было ранних ваншотов.

## 5. Инициатива и первый ход

Инициатива (`_roll_initiative`):
- Игрок: `d20 + perception/2 + luck/3`
- Враг: `d20 + enemy_speed`

Если враг ходит первым, он наносит стартовый удар, но этот удар специально не может убить мгновенно (оставляет минимум 1 HP).

## 6. Формулы боевых статов игрока

Из `models/player.py`:

- `max_health = effective_stamina * 25 + max_health_bonus + hp_upgrade_bonus`
- `melee_damage = 5 + effective_strength + passive_strength_bonus` (+% от артефактов на damage_boost)
- `crit_chance = 5 + (effective_luck - 1) * 2 + artifact_crit + passive_crit`
- `dodge_chance = 10 + passive_dodge + artifact_dodge`
- `total_defense = armor_defense + artifact_defense + passive_defense`

Где `effective_*` учитывают бонусы артефактов.

## 7. Ход игрока (обычная атака)

`handle_combat_attack`:

1. Берется урон оружия (`_resolve_player_weapon`), если оружия нет/пропало — 0.
2. Суммируется с рукопашным: `total_damage = weapon_damage + melee_damage`.
3. Применяется пассивка класса `weapon_damage%` (`_apply_weapon_damage_bonus`).
4. Проверяется крит (`player.crit_chance`):
- множитель крита: `1.5 + crit_damage_bonus/100`.
5. Если враг-`assassin`, он может уклониться (урон становится 0).
6. Нож может наложить кровотечение (3 хода), и тики кровотечения дополнительно снимают HP врага.
7. В конце хода работает реген энергии от выносливости:
- `regen = max(0, (effective_stamina - 4) // 4)`.

## 8. Ход врага и входящий урон

После атаки игрока (если враг жив) враг контратакует:

1. Базовый урон: `combat['enemy_damage']`.
2. Для роли `bruiser` при HP <= 40% включается ярость (`+15% enemy_damage`).
3. Проверка уклонения игрока:
- обычный dodge шанс
- или `perfect_dodge` от скилла.
4. Если попадание прошло:
- применяются защитные эффекты (`temp_defense`, `incoming_damage_reduction`, `enemy_damage_reduction`)
- итог: `final_damage = max(1, enemy_damage - total_defense)`
- урон минимум 1 всегда проходит.
5. Для роли `controller` есть шанс высосать энергию (случайный диапазон).
6. Если HP игрока <= 0: экран поражения -> штрафы -> респавн в больнице.

## 9. Скиллы и кулдауны

Скиллы доступны по классу (или по классу, определенному по оружию).

Механика:
- тратят энергию
- ставят кулдаун в `_skill_cooldowns`
- эффекты хранятся в `_active_skill_effects`
- после хода вызывается `_decrease_cooldowns`

Базовые классы (fallback) и активки:
- Пистолетчик / Снайпер / Дробовик: `Точный выстрел` (+50% к следующей атаке)
- Автоматчик: `Бронирование` (временная защита)
- Пулемётчик: `Заградительный огонь` (задуман как снижение входящего урона)
- Боец: `Уклонение` (`perfect_dodge` на следующую атаку врага)

## 10. Победа, награда, прогресс

`_handle_victory`:

- Опыт и деньги зависят от `reward_mult` врага:
  - `experience = max(5, int(rand(10..30) * reward_mult))`
  - `money = max(3, int(rand(5..25) * reward_mult))`
- Выпадают гильзы (с ограничением вместимости мешочка).
- Обновляются daily-квесты (убийства/гильзы).
- Проверяется левел-ап:
  - при повышении восстанавливаются HP и энергия
  - случайно +1 к одной из характеристик (`strength/stamina/perception/luck`).

## 11. Смерть и последствия

`_handle_death`:

1. Показывается отдельный экран поражения (кто убил, причина, последний урон).
2. Штраф:
- `-50%` денег
- `-25%` опыта
3. Восстановление:
- HP до 50% от максимума
- энергия 50
- радиация сбрасывается в 0
- локация принудительно: `больница`
- если смерть в коридоре, путь сразу прерывается.

## 12. Побег из боя

`handle_combat_flee`:
- Успех: фиксированный шанс 50%.
- Провал: враг наносит удар по обычной формуле входящего урона.
- При смертельном уроне срабатывает стандартная смерть с причиной "погиб при попытке отступления".

## 13. Важные нюансы текущей реализации

1. `Заградительный огонь` (`aoe_damage_reduction`) записывается в активные эффекты, но в текущем одноцелевом расчете урона напрямую не используется.
2. В `handle_combat_attack` есть путь, где при смерти используется `final_damage`; он вычисляется в блоке попадания врага (логически корректно при нормальном ходе, но это место чувствительно к рефакторингу).
3. Входящий урон всегда минимум 1, даже при очень высокой защите.
4. Открывающий удар по инициативе не может убить мгновенно (оставляет минимум 1 HP), а обычные удары в ходе боя могут убить.

## 14. Коротко: почему бой ощущается именно так

- Враги масштабируются под уровень игрока, а не живут фиксированными статами зоны.
- Роли (`танк/хищник/контролёр`) создают разные паттерны угроз.
- Защита, криты, уклонение и классовые пассивки реально участвуют в формулах каждого хода.
- Темп боя задают энергия + кулдауны + шанс побега 50%.
- Коридор/ивенты могут прерывать путь и вставлять бой в общий цикл исследования.
